## Step-by-Step Plan

### Step 1: Replace `src/app/globals.css`

**Action**: Full file replacement with Amol Tracker's CSS-first design system.

Key content:
- `@import "tailwindcss"` (Tailwind v4 syntax — correct)
- CSS custom properties for emerald/gold color palette

**Edge cases**:
- The existing `@theme` block and dark mode variables will be removed. Any code in `src/lib/utils/` or `src/types/` that references these will NOT break since they don't use them.
- The `@custom-variant dark` directive is removed — no dark mode needed for this landing page.

### Step 2: Replace `src/app/layout.tsx`

**Action**: Full file replacement with Amol Tracker layout.

Key changes from existing:
- Replace Geist fonts with Inter (variable font, `--font-inter` variable)
- Replace Geist_Mono — not needed
- Keep Vercel `<Analytics />` and `<SpeedInsights />` (move outside `<main>`)
- Add Google Fonts preconnect links in `<head>` for Hind Siliguri + Inter
- Update metadata: title, description, OpenGraph, Twitter cards
- Set `lang="bn"` on `<html>` (Bengali-first)
- Wrap `{children}` with `<LanguageProvider>`
- Remove the inline dark-mode localStorage script (not needed)
- Remove the skip-to-content link (will be added properly in Chunk 2 with Navbar)

**Font handling (critical)**:
- `Inter` is a variable font — use `subsets: ['latin']`, `variable: '--font-inter'`, `display: 'swap'`
- `Hind Siliguri` is NOT a variable font — need explicit `weight` array for Bengali subset
- Since `Hind Siliguri` may not have a `latin` subset, load it via Google Fonts `<link>` in `<head>` instead of `next/font` to avoid preload warnings
- The CSS `--font-bengali` variable in globals.css will reference the Google Fonts-loaded Hind Siliguri

**Hydration safety**:
- `suppressHydrationWarning` on `<html>` — already present, keep it
- Language class toggling happens in `LanguageContext` via `useEffect` (client-side only) — no hydration mismatch

### Step 3: Create `src/lib/content.ts`

**Action**: Create new file with all bilingual content.

- Pure TypeScript, no React — can be a server-safe module
- `as const` satisfies for type safety
- `Lang` type: `'bn' | 'en'`
- Content object with nested sections: nav, hero, stats, features, screenshots, about, download, footer
- Each string value is `{ bn: string; en: string }` tuple
- Content typed as `Content` for downstream inference

**Edge cases**:
- `as const` on the content object — TypeScript will infer literal types. Downstream `t()` function accepts `{ bn: string; en: string }` — the inferred literal types are assignable to this, so no type errors.
- Bengali Unicode strings (emoji + Bengali script) — no encoding issues with UTF-8.

### Step 4: Create `src/context/LanguageContext.tsx`

**Action**: Create new file with language context provider.

- `'use client'` directive at top
- `LanguageContext` with default values for SSR safety
- `LanguageProvider` component:
  - `useState<Lang>('bn')` — Bengali default
  - `useEffect` to read `localStorage('amol_lang')` on mount
  - `useEffect` to sync: set `document.documentElement.lang`, toggle `lang-bn` class on body, save to localStorage
  - `toggleLang` function: flips between 'bn' and 'en'
  - `t` function: takes `{ bn: string; en: string }`, returns current language string
- `useLang` hook for consuming components

**Edge cases**:
- SSR/hydration: Initial state is 'bn' (matches `<html lang="bn">`). The `useEffect` runs after hydration, so if localStorage has 'en', it changes after mount — this is acceptable and expected (like theme toggles).
- `document.body.classList.toggle('lang-bn', lang === 'bn')` — uses boolean second arg to toggle correctly.
- The `t()` function signature uses `{ bn: string; en: string }` — this works with `as const` inferred types because `string` is a supertype of literal string types.

## Edge Cases & Considerations

1. **Font loading**: Hind Siliguri loaded via Google Fonts `<link>` in `<head>` — avoids `next/font` preload warnings for non-latin subsets. Inter loaded via `next/font/google` with CSS variable.

2. **Hydration**: `LanguageContext` initializes with 'bn' matching `<html lang="bn">`. localStorage read happens in `useEffect` (post-hydration). No mismatch.

3. **Existing code preservation**: `src/lib/utils/index.ts` (cn helper) and `src/types/index.ts` are NOT modified. The `@/*` path alias in tsconfig.json already maps correctly.

4. **Tailwind v4 compatibility**: The new `globals.css` uses `@import "tailwindcss"` (correct for v4). Custom properties are defined in `:root` (not in `@theme` block) since Tailwind v4's `@theme` is for Tailwind utility integration, not raw CSS variables.

5. **Analytics/SpeedInsights**: Preserved from existing layout. Moved outside `<main>` but inside `<body>` for proper rendering.

6. **Google Fonts link in head**: Next.js 16 allows `<link>` tags in root layout `<head>` for external resources. The preconnect hints + stylesheet link are standard practice.
