# Performance Optimization Plan: Carousel & Scroll Smoothness

## Overview

Fix performance issues causing janky carousel scrolling and body scroll interference. The project is a Next.js 16 marketing landing page with a custom carousel, multiple scroll listeners, and inline styles throughout.

---

## Critical Issues to Fix

### 1. Navbar Scroll Listener Re-registration (Critical)

**File:** `src/components/Navbar.tsx` (lines 22-30)

**Problem:** The `useEffect` depends on `[lastY]`, causing the scroll listener to be torn down and re-attached on every scroll event. This creates brief windows where scroll events are missed and causes unnecessary garbage collection pressure.

**Current Code:**
```typescript
useEffect(() => {
  const onScroll = () => {
    const y = window.scrollY;
    setShow(y < lastY || y < 80);
    setLastY(y);  // This triggers re-render and re-attaches listener
  };
  window.addEventListener('scroll', onScroll, { passive: true });
  return () => window.removeEventListener('scroll', onScroll);
}, [lastY]);  // BAD: depends on lastY
```

**Fix:** Use `useRef` to store `lastY` instead of state, and use `useCallback` for the handler:
```typescript
const lastYRef = useRef(0);

const onScroll = useCallback(() => {
  const y = window.scrollY;
  setShow(y < lastYRef.current || y < 80);
  lastYRef.current = y;
}, []);

useEffect(() => {
  window.addEventListener('scroll', onScroll, { passive: true });
  return () => window.removeEventListener('scroll', onScroll);
}, [onScroll]);
```

### 2. Global `scroll-behavior: smooth` Interference (Critical)

**File:** `src/app/globals.css` (line 28)

**Problem:** Global `scroll-behavior: smooth` on `html` interferes with programmatic scrolling and can cause unexpected smooth scrolling behavior. It also affects scroll restoration on back/forward navigation.

**Fix:** Remove global smooth scroll and add it selectively where needed:
```css
/* Remove this: */
html {
  scroll-behavior: smooth;
}

/* Add to specific elements that need smooth scroll */
html.nav-smooth {
  scroll-behavior: smooth;
}
```

Then toggle the class in Navbar when smooth scrolling is needed (for anchor navigation).

### 3. Carousel Drag Performance (Critical)

**File:** `src/components/Screenshots.tsx` (lines 81-111)

**Problem:** Drag-to-scroll handlers (`handleMouseDown`, `handleMouseMove`, `handleMouseUp`, `handleMouseLeave`) are recreated every render without `useCallback` wrapping.

**Fix:** Wrap all drag handlers with `useCallback` and optimize:
```typescript
const handleMouseDown = useCallback((e: React.MouseEvent) => {
  if (!scrollRef.current) return;
  setIsDragging(true);
  setStartX(e.pageX - scrollRef.current.offsetLeft);
  setScrollLeft(scrollRef.current.scrollLeft);
  scrollRef.current.style.cursor = "grabbing";
}, []);

const handleMouseMove = useCallback((e: React.MouseEvent) => {
  if (!isDragging || !scrollRef.current) return;
  e.preventDefault();
  const x = e.pageX - scrollRef.current.offsetLeft;
  const walk = (x - startX) * 1.5;
  scrollRef.current.scrollLeft = scrollLeft - walk;
}, [isDragging, startX, scrollLeft]);

const handleMouseUp = useCallback(() => {
  setIsDragging(false);
  if (scrollRef.current) {
    scrollRef.current.style.cursor = "grab";
  }
}, []);

const handleMouseLeave = useCallback(() => {
  if (isDragging) {
    setIsDragging(false);
    if (scrollRef.current) {
      scrollRef.current.style.cursor = "grab";
    }
  }
}, [isDragging]);
```

### 4. Carousel GPU Optimization (High)

**File:** `src/components/Screenshots.tsx` (CSS in style block)

**Problem:** No GPU layer hints for the carousel, causing browser to composite on every scroll frame.

**Fix:** Add `will-change` and GPU acceleration hints:
```css
.screenshots-scroll {
  will-change: scroll-position;
  -webkit-overflow-scrolling: touch;
}

.screenshot-card {
  will-change: transform;
}

.screenshot-phone {
  will-change: transform;
}
```

---

### 5. Scroll Snap Performance (High)

**File:** `src/components/Screenshots.tsx` (line 365)

**Problem:** `scroll-snap-type: x mandatory` can cause janky snapping on low-end devices because it forces the browser to snap to discrete positions.

**Fix:** Change to `proximity` for smoother behavior:
```css
.screenshots-scroll {
  scroll-snap-type: x proximity;  /* Changed from mandatory */
}
```

### 6. Navbar Throttling (Medium)

**File:** `src/components/Navbar.tsx`

**Problem:** Navbar scroll handler fires on every scroll event without throttling.

**Fix:** Add a simple throttle using `requestAnimationFrame`:
```typescript
const rafRef = useRef<number | null>(null);

const onScroll = useCallback(() => {
  if (rafRef.current) return;
  rafRef.current = requestAnimationFrame(() => {
    const y = window.scrollY;
    setShow(y < lastYRef.current || y < 80);
    lastYRef.current = y;
    rafRef.current = null;
  });
}, []);
```

### 7. Dynamic Import Framer Motion (Medium)

**File:** `src/components/Features.tsx`

**Problem:** `framer-motion` is imported statically but only used in one component, adding unnecessary bundle size.

**Fix:** Use `next/dynamic` to lazy-load Features component:
```typescript
// In page.tsx
import dynamic from 'next/dynamic';

const Features = dynamic(() => import('@/components/Features'), {
  loading: () => <div style={{ minHeight: 400 }} />,
});
```

---

### 8. Image Lazy Loading Optimization (Medium)

**File:** `src/components/Screenshots.tsx`

**Problem:** First 4 images have `priority={true}`, but they're not all above the fold.

**Fix:** Only set priority on the first 1-2 images, and add lazy loading:
```typescript
<Image
  src={`/images/${SCREENSHOT_FILES[i]}.png`}
  alt={t(caption)}
  fill
  style={{ objectFit: "cover" }}
  sizes="220px"
  priority={i < 2}  // Only first 2 images
  loading={i < 2 ? "eager" : "lazy"}
  onError={() => handleError(i)}
/>
```

### 9. CSS Performance Hints (Low-Medium)

### 11. Modal Performance (Low)

**File:** `src/components/Screenshots.tsx` (line 509)

**Problem:** Modal uses `backdrop-filter: blur(16px)` which is expensive.

**Fix:** Reduce blur or use solid background:

.screenshot-modal-overlay {

### 12. Footer Scroll Handler (Low)

**File:** `src/components/Footer.tsx` (line 17)

**Problem:** Simple scroll check, but could be optimized with `requestAnimationFrame`.

**Fix:** Add RAF throttling:
```typescript
const rafRef = useRef<number | null>(null);

useEffect(() => {
  const onScroll = () => {
    if (rafRef.current) return;
    rafRef.current = requestAnimationFrame(() => {
      setShowScrollTop(window.scrollY > 400);
      rafRef.current = null;
    });
  };
  window.addEventListener('scroll', onScroll, { passive: true });
  return () => {
    window.removeEventListener('scroll', onScroll);
    if (rafRef.current) cancelAnimationFrame(rafRef.current);
  };
}, []);
```
