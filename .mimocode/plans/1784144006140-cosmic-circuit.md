# Hero Section Layout Redesign

## Problem Analysis

1. **Nav alignment mismatch**: Hero `paddingTop: 100` doesn't account for fixed navbar height (56px mobile), causing inconsistent spacing
2. **Mobile centering inconsistency**: Grid switches to `text-align: center` but Google Play button has `alignSelf: 'flex-start'` — misaligned
3. **Google Play button vertical layout**: Text stacked vertically (column flex) — takes up too much vertical space, feels awkward
4. **Spacing too generous on mobile**: `gap: 48px` between text and phone mockup is excessive

## Changes — Single File: `src/components/Hero.tsx`

### 1. Add CSS class hooks for mobile targeting
- `hero-container` on the section-container wrapper div (line 68)
- `hero-cta-container` on the CTA flex column (line 129)
- `hero-google-btn` on the Google Play `<a>` (line 137)

### 2. Compact the Google Play button
- Icon: reduce from 28×32 to 24×28
- Padding: reduce from `14px 28px` to `12px 20px`
- "Google Play" text: reduce from `fontSize: 20` to `fontSize: 18`
- "Get it on" text: keep `fontSize: 11`

### 3. Update the `<style>` block

**Desktop** (add to existing base rules):
```css
.hero-container { padding-top: 100px; padding-bottom: 60px; }
```

**Mobile** (`@media max-width: 768px`):
```css
.hero-container {
  padding-top: 72px !important;  /* 56px nav + 16px gap */
  padding-bottom: 40px !important;
}
.hero-grid {
  grid-template-columns: 1fr !important;
  gap: 32px !important;          /* was 48px */
  text-align: center;
}
.hero-headline { font-size: 36px !important; }
.hero-subheadline { font-size: 16px !important; max-width: 100% !important; }
.hero-phone-frame { width: 200px !important; }  /* was 240px */
.hero-phone-col { order: 2; }
.hero-cta-container { align-items: center !important; }
.hero-google-btn { align-self: center !important; }
```

## Decisions Confirmed
- Phone mockup: stays below text on mobile (order: 2)
- Google Play button: center-aligned on mobile
