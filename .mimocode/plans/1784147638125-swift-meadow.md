# Plan: Update Terms & Privacy Routes with Full Content

## Context

The project already has two routes at `/terms` and `/privacy` with simplified content in `TermsPage.tsx` and `PrivacyPage.tsx`. The user has full, detailed HTML content in `TERMS.md` and `PRIVACY_POLICY.md` files and wants these routes to display the complete content instead of the abbreviated versions.

The existing components use inline styles with CSS variables (`--emerald-deep`, `--gold-light`, `--white`). The new content uses its own HTML/CSS styling (green theme with `#0d3d2e` and `#e8c96a` colors).

## Approach

Replace the content inside `TermsPage.tsx` and `PrivacyPage.tsx` with the full content from the MD files, adapted as React components using the same dark green theme as the existing site.

## Files to Modify

1. **`src/components/TermsPage.tsx`** — Replace simplified content with full 14-section content from `TERMS.md` (English only)
2. **`src/components/PrivacyPage.tsx`** — Replace simplified content with full 10-section content from `PRIVACY_POLICY.md` (English only)

## Implementation Details

### TermsPage.tsx
- Keep the existing page shell (back button, header, last-updated badge)
- Update "Last updated" text to "June 2026"
- Replace English content with the 14 sections from TERMS.md: About, Eligibility, User Account, Acceptable Use, Community Features, Amal Data, Islamic Content Disclaimer, Intellectual Property, Availability, Limitation of Liability, Privacy, Changes, Governing Law, Contact
- Remove Bengali content (English only) — always render English
- Use the same inline style pattern (h2/h3/p/ul/li/card/highlight/warning) but styled with CSS variables to match the dark theme

### PrivacyPage.tsx
- Keep the existing page shell
- Update "Last updated" text to "June 2026"
- Replace English content with the 10 sections from PRIVACY_POLICY.md: Information Collected, How We Use, Community Data, Data Storage, Retention, Third-Party Services, Children's Privacy, Your Rights, Changes, Contact
- Remove Bengali content (English only) — always render English
- Include the third-party services table

### Content Structure
Each page will use typed React components with consistent style objects:
- `sectionTitle` — h2 style (gold, bold, border-bottom)
- `subTitle` — h3 style (lighter gold)
- `bodyText` — p style (white 70% opacity, 1.8 line-height)
- `listItem` — li style
- `card` — white/dark card container
- `highlight` — green accent box
- `warning` — red accent box
- `contactBox` — dark green contact section
